Log. Prototype Guidelines
Prototype Guidelines

Rules for creating, versioning, and graduating Log• prototypes. The system is the source of truth. Prototypes explore within it, not around it.

Prototype Territory
May change
  • Layout and content density
  • Navigation flow between surfaces
  • Copy and placeholder text
  • Sample data and entry content
  • Prototype-specific layout (device frames, stage backgrounds)
  • Number of items in lists or grids
  • Order of elements within a surface
  • Interaction patterns (swipe vs. long-press)
  • Prose in archivist annotations
Must not change
  • Color tokens (paper tones, signal colors, amber)
  • Signal colors in dark mode — they do not invert
  • Font families: Space Mono = mechanism, IBM Plex Sans = tape
  • Spacing grid (8pt — use CSS variables only)
  • Intent dot size: 6px, border-radius 50%
  • Idle trace: no animation, instant at final height
  • Touch targets: 44px minimum
  • Icon stroke: 1.5px at 24px canvas, currentColor
  • The mark: "Log" + amber period, Space Mono 600
  • No emoji in UI — SVG only
  • Dark mode: 100% coverage before presentation
Naming Convention
[name]-surface-v[N].html   or   [name]-behavior.html
  • timeline-surface-v1.html → timeline-surface-v2.html
  • entry-detail-surface-v1.html
  • search-surface-v1.html
  • idle-trace-behavior.html
  • dot-classification-behavior.html

Surface prototypes are device-framed UI mockups. Behavior prototypes demonstrate a specific interaction pattern without a device frame. Both types live in prototypes/.

Versioning

Increment N for each new version: timeline-surface-v1.html → timeline-surface-v2.html. Never overwrite an existing version — old versions are the record.

Behavior files do not version (they document rules, not designs). If a rule changes, update the behavior file in place and note the change in a comment.

The working version is always the highest N. If multiple versions exist, the oldest may be archived to prototypes/archive/ but never deleted.

Graduation Criteria

A prototype graduates to a system component when all five are true:

  1. Dark mode complete

    Every element has a verified dark mode state. No paper-40 text on paper-90/95 backgrounds.

  2. Tokens only

    No hardcoded hex values. All colors, spacing, and type sizes reference CSS custom properties from the shared stylesheet.

  3. Accessibility verified

    Touch targets at 44px minimum. Text contrast passes WCAG AA. SVG icons have aria-labels. Interactive elements have focus states.

  4. Specifier reviewed

    A specifier pass has produced a formal component specification with all states, measurements, and behavior rules documented.

  5. Ethos gate passed

    All five calibration questions answered without uncertainty: necessary, coherent, simplest correct, nothing to remove, consistent with system.
